第 4 关

选主策略

ODOWN 确认！现在从 Slave-1 和 Slave-2 中选出新 Master。
两个从节点状态如下：

节点 slave_priority 复制偏移量 offset Run ID（字典序）
Slave-1 100 102400 3a8e...
Slave-2 100 108920 1c2f...
选主顺序：① slave_priority 越小越优先 → ② 复制偏移量越大越优先（数据越新）→ ③ Run ID 字典序越小越优先

哪个从节点会被选为新 Master？

Slave-1（Run ID 字典序更大）
Slave-2（复制偏移量更大，数据更新）
随机选一个

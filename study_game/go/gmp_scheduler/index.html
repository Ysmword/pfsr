第 1 关

G / M / P 是什么？

Go 调度器由三种实体组成。把右边的标签拖到对应的定义框中：

G
M
P
Goroutine — 用户态轻量线程，创建成本极低（初始栈 ~2KB），Go 程序的并发执行单元
Machine / OS Thread — 操作系统线程，由 OS 调度，数量通常等于 CPU 核数
Processor / 逻辑处理器 — 持有本地 G 队列 + 调度上下文，M 需绑定 P 才能执行 G

核心关系：M 必须绑定 P 才能执行 G。
GOMAXPROCS 控制 P 的数量（默认=CPU 核数），决定最大并行度。
G 的数量可以远远多于 M/P 数量，这就是 Go 高并发的基础。
